# אלגוריתמים — לראות בעיניים

מערכת אינטראקטיבית ללימוד הקורס **"מבני נתונים ומבוא לאלגוריתמים"**, ויזואלית ובעברית.

### 🔗 הממשק החי: **https://maozepstein.github.io/algo/**

קוד המקור: [github.com/MaozEpstein/algo](https://github.com/MaozEpstein/algo)

---

## 🎯 המטרה

ללמד אלגוריתמים **בראייה ולא בקריאה**. במקום קירות טקסט — רואים את האלגוריתם רץ צעד-אחר-צעד,
עם הקוד שמתרחש לצידו, אנימציות שמראות מה קורה למבנה הנתונים, והאינטואיציה מאחורי כל מהלך.
הדגש הוא על **"לראות את זה"** — כך שגם מושגים מופשטים (סיבוכיות, רקורסיה, נוסחאות נסיגה)
הופכים מוחשיים. הממשק כולו בעברית (RTL); הפסאודו-קוד וה-Python נשארים באנגלית, כמו בקורס.

## ✨ היכולות

- **ויזואליזציה מודרכת** — נגן עם הרצה/השהיה/דילוג צעד-צעד וקיצורי מקלדת, עם **הדגשת שורת הקוד**
  הרצה (פסאודו-קוד + מתג ל-Python).
- **תצוגה כפולה** מערך ↔ עץ בינארי, עם אנימציות FLIP — הזהות נוסעת עם הערך, כך שהחלפות "זזות".
- **מוני עלות** (השוואות והחלפות) שמתעדכנים תוך כדי ריצה.
- **קלט ניתן לעריכה + דוגמאות מוכנות** (presets) של מקרי-קצה ומקרים גרועים בלחיצה.
- **הוכחות סיבוכיות** מובנות (טענה → צעדים → אינטואיציה), תגי "ראשי/עזר", וגרף קריאות.
- **מצב סיכום** עם KaTeX, טבלאות סיבוכיות וייצוא ל-PDF.
- **מבט-על בין-שיעורי** — טבלת השוואת מיונים ו"מירוץ אלגוריתמים" (שני אלגוריתמים על אותו קלט).
- **שיעורי "מסביר"** אינטראקטיביים לניתוח (ולא להרצת אלגוריתם) — מנחשי-חסם, פיתוח נוסחאות
  צעד-צעד, עצי-עלות נבנים, ומחשבון אינטראקטיבי לשיטת האב.
- **קישורים עמוקים** (deep-links) לכל שיעור/מצב/לשונית, ואתר סטטי ניתן לשיתוף.

| שיעור | נושא | מה מומש |

| **3 · חלק ב׳** | נוסחאות נסיגה<br>(Recurrence Relations) | שיעור-מסביר ב-4 לשוניות: **מבוא** (הגדרה, טבלת "נוסחאות שכדאי להכיר", מפת-דרכים בין השיטות) · **שיטת ההצבה** (5 דוגמאות עם מנחש-חסם והוכחה מלאה, כולל חציון-של-חציונים ופיבונאצ׳י) · **שיטת האיטרציה** (פיתוח צעד-צעד עם עץ-עלות / מדרגות שנבנים בסנכרון) · **שיטת האב** (מחשבון אינטראקטיבי, 15 דוגמאות מסווגות ל-3 המקרים) |

| **5** | מיון מהיר<br>(Quicksort) | Hoare-Partition · Quicksort · Lomuto-Partition · Randomized-Quicksort — תצוגת מערך |

## 🏗️ ארכיטקטורה

הליבה היא **מנוע פריימים**: כל אלגוריתם הוא גנרטור דטרמיניסטי טהור שמפיק `Frame[]`
(ראו `src/engine/`). כל המצבים צורכים את אותה רשימת פריימים; הזהות נוסעת עם הערך
(`elementId`), לא עם התא — זה מה שמאפשר לאנימציית ההחלפה לעבוד.

- `src/engine/` — טיפוסים, `FrameBuilder`, חשבון אינדקסים (1-indexed, CLRS).
- `src/lectures/<id>/` — מודול שיעור עצמאי המממש את `LectureModule` (אלגוריתמים + פסאודו-קוד + סיכום,
  או שיעור-**מסביר** עם תוכן-לשוניות משלו).
- `src/viz/` — התצוגה הכפולה מערך↔עץ (SVG + Framer Motion).
- `src/shell/`, `src/modes/` — מעטפת, נגן ופאנלים — אגנוסטיים לשיעור.

**הוספת שיעור חדש** = ליצור `src/lectures/<id>/` שמייצא `LectureModule`, ולהוסיף שורה אחת
ב-`src/app/registry.ts`. כל השאר (מצבים, נגן, RTL, deep-links, כרטיס בדף הבית) מגיע חינם.

## ☁️ פריסה

כדי ש-deep-links ישרדו רענון. (קיימים גם `vercel.json` ו-`public/_redirects` לפריסה ב-Vercel/Netlify.)
